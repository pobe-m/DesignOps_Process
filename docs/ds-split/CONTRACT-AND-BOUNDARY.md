# Contract + Boundary — ทำให้ "Regenerate ห้าม Rewrite Business Logic" บังคับได้จริง (ข้อ 8)

> ปัญหาหลัก: ถ้า blueprint กับ business logic อยู่ไฟล์เดียวกัน การ regenerate **จะทับ logic ทิ้งแน่นอน**.
> ข้อ 8 จะเป็น guarantee ได้ก็ต่อเมื่อมี **seam ทางกายภาพ** + **enforcement ที่เครื่องตรวจได้** ไม่ใช่ข้อตกลงในเอกสาร

## หลักการ 3 ชั้น

```
1. แยกที่อยู่ (Directory boundary)  — generated/ vs src/
2. แยกผ่าน contract (Contract seam) — generated พูดกับ src ผ่าน interface คงที่เท่านั้น
3. บังคับด้วยเครื่อง (Enforcement)   — CODEOWNERS + CI guard + codegen markers
```

---

## 1. Directory Boundary

โครงสร้าง product repo ที่ scaffold สร้าง:

```
product-repo/
├── generated/                 ← DesignOps เขียน/ลบ/regenerate ได้ 100%
│   ├── blueprints/            ← screen blueprint (layout, props, data shape)
│   ├── screens/               ← UI shell ที่ประกอบจาก @acme/ds
│   └── contracts/             ← interface ที่ generated เปิดให้ src ใช้
│
├── src/                       ← Dev team เท่านั้น (ข้อ 7). bot แตะไม่ได้
│   ├── logic/                 ← business logic, rules, calculations
│   ├── services/              ← API, data fetching, mutations
│   └── bindings/              ← เชื่อม generated screen ↔ src logic ผ่าน contract
│
├── brand.config.json          ← อยู่ product repo (ข้อ 6), dev แก้ได้
├── CODEOWNERS                 ← บังคับ ownership
└── package.json               ← import @acme/ds (ข้อ 5)
```

**กฎเหล็ก:** generated/ **ห้าม import จาก src/** และ src/ คุย generated/ ผ่าน `generated/contracts/` เท่านั้น
→ regenerate ลบ `generated/` ทั้งก้อนแล้วสร้างใหม่ได้ โดย `src/` ไม่รู้สึกอะไร ตราบใดที่ contract เดิม

## 2. Contract Seam — interface 3 ระดับ

### 2.1 Component Contract (อยู่ใน DS repo)
prop schema ของ 52 component — สิ่งที่ blueprint อ้างถึงได้ ห้ามเกินนี้

```jsonc
// @acme/ds/contracts/button.contract.json
{
  "component": "Button",
  "import": "@acme/ds",
  "props": {
    "variant": { "enum": ["default","outline","secondary","ghost","destructive","link"] },
    "size":    { "enum": ["xs","sm","md","lg","default"] },
    "disabled":{ "type": "boolean" }
  },
  "slots": ["children"],
  "stable_since": "1.0.0"
}
```
→ blueprint generate ได้เฉพาะ prop ที่อยู่ใน contract = upgrade DS ไม่พัง blueprint แบบเงียบๆ

### 2.2 Screen Blueprint (DesignOps emit → generated/blueprints/)
สิ่งที่ DesignOps **เป็นเจ้าของเต็ม** — regenerate ได้อิสระ

```jsonc
// generated/blueprints/booking.blueprint.json
{
  "screen": "Booking",
  "layout": "two-column",
  "regions": [
    { "slot": "form", "components": ["Input","DatePicker","Button"] }
  ],
  "data_contract": "BookingData",        // ← ชี้ไปที่ contract, ไม่ใช่ logic
  "actions": [{ "name": "submitBooking", "contract": "BookingActions" }]
}
```

### 2.3 Data/Action Contract (seam จริงระหว่าง generated ↔ src)
จุดตัดที่ทำให้ข้อ 8 เป็นไปได้ — generated รู้แค่ "รูปร่าง" ไม่รู้ "logic"

```ts
// generated/contracts/booking.ts   ← generated เขียน (type-only, ไม่มี logic)
export interface BookingData { date: string; guests: number; roomId: string }
export interface BookingActions {
  submitBooking(data: BookingData): Promise<{ ok: boolean; id?: string }>
}
```
```ts
// src/bindings/booking.ts           ← DEV เขียน logic จริงที่นี่ (bot ห้ามแตะ)
import type { BookingActions } from "@/generated/contracts/booking"
export const bookingActions: BookingActions = {
  async submitBooking(data) { /* ...business logic, validation, pricing... */ }
}
```

generated screen รับ actions ผ่าน prop/context → **regenerate UI shell ได้ตลอด โดย submitBooking ของ dev ไม่ถูกแตะ**

## 3. Enforcement — บังคับด้วยเครื่อง (ไม่พึ่งวินัยคน)

### 3.1 Codegen markers
ทุกไฟล์ใน `generated/` ขึ้นต้นด้วย:
```ts
// @generated by designops — DO NOT EDIT. Changes are overwritten on regenerate.
// source: blueprints/booking.blueprint.json
```

### 3.2 CODEOWNERS (ข้อ 7 + 8)
```
/generated/   @org/designops-bot
/src/         @org/dev-team
/brand.config.json  @org/dev-team
```

### 3.3 CI guard (หัวใจของการบังคับ)
- PR จาก designops-bot ที่แตะไฟล์ **นอก** `generated/` → **CI fail**
- ไฟล์ใน `generated/` ที่ถูกแก้ด้วยมือ (ไม่ตรง marker/hash) → CI เตือน
- regenerate = เปิด PR เท่านั้น (Phase 6) ไม่ push ตรง

```bash
# .github/workflows/guard.yml (pseudo)
changed=$(git diff --name-only origin/main)
if [ "$PR_AUTHOR" = "designops-bot" ] && echo "$changed" | grep -qv '^generated/'; then
  echo "::error::designops PR touched files outside generated/"; exit 1
fi
```

## ทำไมแค่ "แยกโฟลเดอร์" ไม่พอ

| แนวทาง | ปัญหา | บังคับข้อ 8 ได้? |
|---|---|---|
| แยกโฟลเดอร์เฉยๆ | คนยังเขียน logic ปนใน generated/ ได้ | ❌ |
| Codegen markers อย่างเดียว | คนลบ marker แล้วแก้ได้ | ⚠️ บางส่วน |
| **Directory + Contract + CODEOWNERS + CI guard** | bot แตะ src/ ไม่ได้, generated regenerate ได้อิสระ | ✅ |

**สรุป:** ข้อ 8 = **Directory boundary + Contract seam + CI guard** ครบ 3 อย่าง. ขาดอันใดอันหนึ่งกลายเป็น "ความตั้งใจดี" ที่ regenerate ครั้งหน้าก็ทับ logic อยู่ดี
